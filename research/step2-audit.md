# Gewu（格物）第二步开发审计报告
日期：2026-03-26

## 1) 当前状态
当前已完成第二步的可运行 MVP，并补齐了一批高优先级问题修复：

- 已完成三端基础框架：`web`、`admin`、`api`、`worker`
- 已完成用户工作台左侧导航的后台可控（显示/隐藏）
- 已新增管理员接口保护（`x-admin-token`）
- 已完成密码哈希校验（`scrypt`）
- 已完成工作台“路由级”导航拦截（隐藏后不可直链访问）
- 已新增编码异常自动检测脚本，防止乱码回归

## 2) 需求对照
### 2.1 已满足（MVP 级）
- 前台页面：首页、定价、教程、登录、注册
- 工作台页面：`/zh/*` 主要功能路由均已创建
- 后台页面：Dashboard、用户、任务、订单、套餐、模型、内容、积分、系统设置
- 模型路由模块：`ModelRouter` 已接入并包含目标模型清单
- 队列框架：BullMQ API 与 Worker 已搭建
- 新增能力：后台可控制工作台左侧导航项显隐

### 2.2 部分满足
- Google OAuth：仅保留入口，未做真实联调
- 后台部分 CRUD：仍有占位逻辑，尚未全部落库
- 模型 Key：有开关和状态位，未接入真实密钥托管与供应商联通

### 2.3 尚未满足（后续阶段）
- OSS 文件全链路（上传、处理结果回写、下载鉴权）
- 长文本真实 SSE 流式输出（当前仍以前端模拟为主）
- 支付/充值/计费闭环（按你要求，先做核心功能，支付后补）
- 完整持久化替换（当前仍有内存态数据）

## 3) 已修复问题清单
### P0/P1 已修复
1. 管理接口未鉴权  
修复：`/api/v1/admin/*` 全部加 `preHandler`，要求 `x-admin-token`

2. 明文密码比较  
修复：新增 `hashPassword/verifyPassword`，登录改为哈希校验，保留旧明文兼容兜底

3. 工作台隐藏菜单可被直链访问  
修复：工作台前端新增路由守卫，隐藏项直链会自动重定向到首个可见页

4. 文案乱码回归风险  
修复：清理乱码文案并新增 `scripts/check-mojibake.mjs`，纳入根 `lint` 流程

## 4) 残余风险与建议
1. 内存态存储仍不适合生产  
建议：优先把用户、积分、任务、模型配置、导航配置迁移到 PostgreSQL（Prisma）

2. 后台 Token 仍为单令牌模式  
建议：后续升级为管理员登录会话 + RBAC + 操作审计

3. Admin 打包体积偏大  
建议：按页面懒加载和图表拆包，优化首屏资源

## 5) 关键接口（本阶段）
- `GET /api/v1/workbench/nav`
- `GET /api/v1/admin/workbench-nav`
- `PATCH /api/v1/admin/workbench-nav/:key`
- `GET /api/v1/admin/models`
- `PATCH /api/v1/admin/models/:id`
- `GET /api/v1/admin/settings`
- `PUT /api/v1/admin/settings`
